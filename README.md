# Omniboost

Omniboost is a web damage calculator workspace for Pokemon Champions / VGC-style doubles. It is built around a fast prompt grammar, client-side damage calculation, and a UI that behaves more like an editor than a traditional calc form.

## What It Does

- Parses compact matchup prompts such as:
  - `politoed !muddy-water @mystic-water x incineroar @assault-vest ~rain`
- Resolves Pokemon, forms, megas, moves, items, abilities, status, HP, field states, and multi-hit move counts
- Calculates damage client-side with 3 default bulk outputs:
  - `Min Bulk`
  - `Mid Bulk`
  - `Max Bulk`
- Supports Pokemon Champions SP-based set editing
- Provides searchable set editing for species, item, ability, moves, nature, and SP spread
- Uses current Champions meta snapshots for autocomplete and defaults

## Prompt Syntax

Canonical structure:

```txt
{attacker} !{move} [attacker tokens...] x {defender} [defender tokens...] [global tokens...]
```

Example:

```txt
politoed !muddy-water @mystic-water x incineroar ~rain
```

Common tokens:

- `!move`
- `#set-name` to reference a saved/shared set by nickname or species id
- `@item`
- `%75` for current HP in that segment
- `sp:32/0/1/13/1/19` for an explicit Champions SP spread in that segment
- `[Ability Name]`
- `~rain`, `~sun`, `~grassy-terrain`, `~trick-room`
- `*` for critical hit
- `+1` to `+6` / `-1` to `-6` for relevant attacking or defending stages
- `spe+1` to `spe+6` / `spe-1` to `spe-6`
- `burn`, `paralysis`, `poison`, `sleep`, `freeze`
- `!bullet-seed(3)` style multi-hit specification

## Features

- Segment-scoped prompt grammar
- Fuzzy Pokemon and move matching
- Searchable quick suggestions and keyboard navigation
- Client-side damage calculation
- Mega form support
- Regulation-aware legal species filtering
- Pokemon summary cards with sprite, stats, item, ability, move, and set editing
- SP-aware set editor for Pokemon Champions
- Shareable calc URLs that can restore prompt + relevant custom sets
- Competitive meta-driven autocomplete data

## Current Data Model

Omniboost does not fetch live competitive data during normal runtime. It uses committed snapshots under `src/data/`, generated by local scripts.

Main runtime data files:

- `src/data/pokemon.gen9.json`
- `src/data/moves.gen9.json`
- `src/data/learnsets.gen9.json`
- `src/data/vgc-meta.json`
- `src/data/form-aliases.json`
- `src/data/regulations/active.json`
- `src/data/regulations/regulation-m-a.json`

- `npm run generate:all` - regenerate all data snapshots

## Data Pipeline

Build-time sources:

- `@pkmn/dex` and `@pkmn/data` for canonical species, moves, and learnsets
- Pikalytics Champions AI pages for competitive usage/meta
- Serebii Champions mega abilities for mega ability corrections
- local regulation files for legal roster enforcement

Runtime external assets:

- Pokemon Showdown sprite CDN
- PokemonDB image fallback URLs

The main aggregation layer is:

- `src/lib/data/loaders.ts`

## Project Structure

Key directories:

- `src/app` - Next.js app entrypoints and global styles
- `src/components` - UI components
- `src/lib` - parser, calc, data loading, grammar, utilities
- `src/store` - Zustand stores
- `src/data` - committed runtime data snapshots and regulation files
- `scripts` - snapshot generation scripts
- `docs` - prompt grammar and data-source documentation

## Notes

- The active regulation is controlled by `src/data/regulations/active.json`.
- Legality for Regulation M-A is maintained in `src/data/regulations/regulation-m-a.json`.
- Meta defaults and suggestions come from generated `src/data/vgc-meta.json`.
- Images are loaded from external sprite/artwork sources at runtime; gameplay/meta data is not.
