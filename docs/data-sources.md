# Unburden Data Sources

This document lists the data sources Unburden uses, what each source owns, and where the app consumes generated snapshots.

## Source-of-truth policy

- Runtime gameplay and meta data is local. The app reads committed JSON snapshots under `src/data/`.
- Runtime network access for Pokemon images is limited to Pokemon Showdown sprite URLs generated by `@pkmn/img`.
- Structural Pokemon Champions data comes from Pokemon Showdown data packaged by `pkmn/ps`.
- Pokemon Champions meta defaults come from Smogon usage statistics for Champions VGC 2026 Reg M-A, normalized into `src/data/vgc-meta.json`.
- Serebii and Pikalytics are not part of the default data flow.

## External sources

### `pkmn/ps` packages

Packages:

- `@pkmn/dex`
- `@pkmn/mods/champions`
- `@pkmn/img`

Used by:

- `scripts/generate-static-data.ts`
- `scripts/transform/build-static-data.ts`
- `src/lib/pokemon-sprites.ts`

Owns:

- Pokemon Champions species/form data
- Pokemon Champions base stats, types, abilities, required mega items, and form relationships
- Pokemon Champions movedex data and Champions-specific move overrides
- Pokemon Champions learnsets
- Pokemon Champions legal held-item pool
- Pokemon Champions Regulation M-A roster derived from Showdown Champions format/species data, with local compatibility fixes when app behavior requires a support species
- Pokemon Showdown sprite URL generation

Outputs:

- `src/data/pokemon.gen9.json`
- `src/data/moves.gen9.json`
- `src/data/learnsets.gen9.json`
- `src/data/champions-items.json`
- `src/data/regulations/regulation-m-a.json`
- `src/data/regulations/active.json`

### Smogon Champions usage statistics

Sources:

- `https://www.smogon.com/stats/YYYY-MM/chaos/gen9championsvgc2026regma-1500.json`
- auto-latest month discovery from `https://www.smogon.com/stats/`

Used by:

- `scripts/generate-vgc-meta.ts`
- `scripts/fetch/fetch-smogon-stats.ts`
- `scripts/transform/build-vgc-meta.ts`

Owns:

- Champions metagame usage order / `usageRank`
- usage percent
- common moves, abilities, and items
- default move, ability, and item derivation inputs

Configuration:

- default source: Smogon raw usage stats
- default format: `championsvgc2026regma` / `gen9championsvgc2026regma`
- default cutoff: `1500`
- default mode: BO1
- default month: `latest`, with optional pinning through `src/data/vgc-meta.overrides.json` or `npm run generate:vgc-meta -- --month YYYY-MM`

Output:

- `src/data/vgc-meta.json`

### Pokemon Showdown sprite CDN

Sources are generated through `@pkmn/img`, currently from Pokemon Showdown sprite resources.

Used by:

- Pokemon summary / identity UI components
- Speed ladder UI

Owns:

- sprite URL generation only

Gameplay data is not fetched at runtime.

## Local config and snapshots

### `src/data/form-aliases.json`

Purpose:

- explicit alias and form resolution
- parser / autocomplete form handling
- Showdown import normalization
- meta generation form mapping

### `src/data/vgc-meta.overrides.json`

Purpose:

- local overrides for generated competitive meta profiles

Effects:

- can override generated defaults, source format/month/cutoff, limits, species mappings, and profile fields

## Practical data flow

### Build time

1. `scripts/generate-static-data.ts` transforms Pokemon Showdown Champions package data into structural snapshots.
2. `scripts/generate-vgc-meta.ts` fetches and transforms Champions usage/meta data from Smogon stats.
3. Generated snapshots are validated for minimum counts, unique ids, legal item references, and large profile deltas before writing.
4. Generated JSON files are committed under `src/data/`.

### Runtime

1. Domain modules under `src/lib/data/` import committed JSON snapshots.
2. Parser, autocomplete, damage calculator, Speed Benchmark, stores, and UI consume in-memory maps/arrays.
3. No live gameplay or meta fetch runs during normal app usage.
4. Pokemon images are loaded from Pokemon Showdown sprite URLs generated by `@pkmn/img`; failed images fall back to a local typed placeholder.

## Troubleshooting stale or missing data

If the app looks stale or a legal Pokemon/item is missing, inspect these first:

- `scripts/generate-static-data.ts`
- `scripts/generate-vgc-meta.ts`
- `scripts/fetch/*`
- `scripts/transform/*`
- `src/data/vgc-meta.overrides.json`
- the generated snapshots under `src/data/`

Runtime UI code should usually not be the first place to change source accuracy.
